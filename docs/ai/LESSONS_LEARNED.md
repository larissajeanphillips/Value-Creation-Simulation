# Lessons Learned

This document captures recurring issues, their root causes, and how to prevent them in the future.

## How to Use

When you encounter a bug or issue that:
- Took significant time to debug
- Was caused by a non-obvious root cause
- Could happen again in a similar context

Add an entry here so future developers (and AI) can learn from it.

---

## Template Entry

### [Category]: [Brief Title] (YYYY-MM-DD)

- **Symptom**: What error/behavior was observed
- **Root cause**: The actual underlying issue
- **Fix**: How it was resolved
- **Prevention**: How to avoid this in the future
- **How to verify**: Steps to confirm the fix works

## Categories

Use these categories for organization:

- **Build**: Compilation, bundling, deployment issues
- **Runtime**: Errors that occur during execution
- **UI**: Visual bugs, layout issues
- **Performance**: Slow operations, memory leaks
- **Data**: Database, API, data processing issues
- **Config**: Environment variables, configuration problems
- **Integration**: Third-party service issues
